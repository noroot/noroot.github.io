Switched from Jekyll to Bridgetown

Category updates
Tags #dev #tools

Long story short, - I‚Äôve decided to switch from Jekyll to Bridgetown. First of all I have enough with Jekyll, I am using it for about 10 years, and its became stiff to change. Last time I‚Äôve tried to do something new with Jekyll I got struggle with simple tasks, maybe I do something wrongway (highly unlikely) but I have no patience to continue with Jekyll and research too much to solve simple things which can be written from scratch in about hour.

So here it is Bridgetown.rb - So it is not only static site generator, its feature rich lightweight framework, which allow you to generate your site statically. It uses Puma as sever, Roda as main routing library. It is also utilizes tools like Rake, and it has integrated deployment feature. All of it makes bridgetownrb highly customizable, friendly to programmers solution for ‚Äúfast and hacky‚Äù projects.

Features:

  • Template engine (erb, liquid, serbea)
  • FrontMatter
  • Layouts
  • Components
  • Collections
  • Categories
  • Tags
  • Assets building
  • Dynamic sites
  • Static generated sites
  • Deployment

How to migrate from Jekyll

So how to migrate ? FrontMatter is a small YAML snippet with metadata for each post on top of the post. It is the same format as Jekyll, Hugo, and many others. All you need to do is copy all posts into the _posts directory.

And because of this YAML snippet on top of the post file, Bridgetown generator will understand everthing and generate same structure as I have in Jekyll

---
layout: post
title: "Switched from Jekyll to Bridgetown"
categories: howto
tags: [dev]
date:   2024-06-22 12:00:00 +0000
---
